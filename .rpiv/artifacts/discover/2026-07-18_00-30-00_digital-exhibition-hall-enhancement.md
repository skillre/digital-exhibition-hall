# FRD: 展厅交互增强与科技极简风格升级

## Summary
将当前基于 Three.js 的数字展厅从"初级静态展示"升级为"成熟、高科技、专业"的科技极简风格，增强交互功能包括：可交互 3D 模型、实时数据可视化、3D 模型嵌入展板、全屏沉浸模式；修复碰撞检测 bug，添加传送与导览、自动巡展模式；并优化视觉效果（后处理、粒子、动画）。

## Problem & Intent
当前数字展厅存在两个核心问题：
1. **视觉效果初级**：展厅太空旷，材质纯色无纹理，无后处理特效，缺乏科技感和高级感
2. **交互功能有限**：仅支持基础的展板 hover/click，缺少多样化的交互对象和内容展示方式

用户希望将展厅升级为面向"客户/领导/合作伙伴"的综合展示平台，需要兼顾专业性和视觉冲击力，给参观者留下"成熟、专业、值得信赖"的印象。

## Goals
- **视觉升级**：实现科技极简风格（白色/浅灰主调 + 蓝色点缀），类似苹果/特斯拉官网的高级感
- **交互增强**：支持可交互 3D 模型、实时数据可视化、3D 模型嵌入展板
- **控制优化**：修复碰撞检测 bug，添加传送点、自动巡展模式
- **内容丰富**：展板支持全屏沉浸模式，提供更清晰的内容查看体验
- **性能稳定**：在增加视觉特效的同时保持 60fps 流畅运行

## Non-Goals
- ❌ 不对接真实后端 API，使用静态 JSON 数据
- ❌ 不支持多人协作/社交功能
- ❌ 不实现移动端触屏适配（本次迭代）
- ❌ 不支持外部 glTF/GLB 模型加载，使用程序化生成

## Functional Requirements

### 1. 视觉风格升级
1.1. 材质系统重构：使用科技极简风配色（主色 #f5f5f7 浅灰、点缀色 #0071e3 蓝色），支持材质纹理
1.2. 后处理效果：添加 Bloom（发光元素辉光）、SSAO（增强阴影层次）
1.3. 粒子系统：入口区域添加光点飘浮效果，营造科技氛围
1.4. 动画效果：展板呼吸灯、进场动画、灯光脉动

### 2. 可交互 3D 模型
2.1. 支持在展厅内放置程序化生成的 3D 模型（安全设备、架构图等）
2.2. 模型支持鼠标拖拽旋转、滚轮缩放
2.3. 模型悬停时显示信息标签
2.4. 点击模型可查看详细说明

### 3. 实时数据可视化
3.1. 展厅内设置数据大屏区域，展示安全态势指标
3.2. 数据从静态 JSON 文件读取，定时模拟更新（每 30 秒）
3.3. 支持折线图、柱状图、饼图等图表类型
3.4. 图表支持悬停查看具体数值

### 4. 展板增强
4.1. 展板内容支持 3D 模型嵌入（程序化生成的几何体组合）
4.2. 点击展板可进入全屏沉浸模式，内容居中放大显示
4.3. 全屏模式支持 ESC 键或点击背景退出

### 5. 控制系统优化
5.1. 修复展板碰撞检测 bug：将展板加入碰撞检测列表
5.2. 优化碰撞响应：实现沿墙滑动效果（分别检测 X/Z 轴）
5.3. 传送系统：小地图点击传送、预设热点快速跳转
5.4. 自动巡展模式：按预设路径自动移动，定时停留展示

### 6. UI 优化
6.1. 小地图增强：显示传送点、当前位置、展区标识
6.2. 操作指南更新：添加新功能说明
6.3. 加载界面：添加加载进度条

## Non-Functional Requirements

### Performance
- **帧率**：在中等配置设备（GTX 1060 级别）保持 60fps
- **加载时间**：初始加载 < 3 秒（不包括模型文件）
- **内存占用**：稳定运行时 < 500MB

### Security
- 无特殊安全要求，静态前端应用

### UX / Accessibility
- **操作反馈**：所有交互都有视觉/音效反馈
- **指引清晰**：新用户首次进入有引导提示
- **响应式**：支持常见分辨率（1920×1080 为主）

### Reliability
- **错误处理**：模型加载失败时显示占位符
- **降级方案**：后处理效果在低端设备自动关闭
- **资源清理**：离开页面时正确释放 GPU 资源

## Constraints & Assumptions
- **技术约束**：基于 Three.js r128+，不引入其他 3D 引擎
- **数据约束**：所有数据来自静态 JSON 文件，无需后端服务
- **模型约束**：使用程序化生成（BoxGeometry、SphereGeometry 等），不加载外部模型
- **浏览器约束**：支持 Chrome、Edge、Firefox 最新版本
- **假设**：用户设备支持 WebGL 2.0

## Acceptance Criteria
- [ ] 展厅整体呈现科技极简风格，配色统一（浅灰 + 蓝色点缀）
- [ ] 展板、门框等发光元素有 Bloom 辉光效果
- [ ] 入口区域有光点飘浮粒子效果
- [ ] 展厅内可放置可旋转的 3D 模型（至少 3 种不同类型）
- [ ] 数据大屏展示至少 4 个实时指标（图表形式）
- [ ] 展板内容支持 3D 模型嵌入（至少 2 个展板有 3D 内容）
- [ ] 点击展板可进入全屏沉浸模式，ESC 退出
- [ ] 玩家无法穿过展板，碰撞时可沿墙滑动
- [ ] 小地图支持点击传送，至少 4 个传送点
- [ ] 自动巡展模式可按预设路径导览，停留 5 秒/展区
- [ ] 在 GTX 1060 级别设备上保持 60fps
- [ ] 首次加载时间 < 3 秒

## Recommended Approach

**视觉升级**：使用 `three/examples/jsm/postprocessing` 添加 EffectComposer 管线，包含 RenderPass + UnrealBloomPass + SSAOPass。材质系统重构为 MeshStandardMaterial 组合（color + roughness + metalness + emissive）。

**交互增强**：创建 `InteractiveModel` 类封装可交互 3D 模型逻辑，`DataDashboard` 类管理数据可视化大屏。展板系统扩展支持 `model3d` 类型和全屏模式。

**控制优化**：修复 `app.js:53` 碰撞列表，添加展板到 walls 数组。`PlayerControls.js` 碰撞响应改为分别检测 X/Z 轴实现滑动。新增 `TeleportSystem` 和 `AutoTourMode` 模块。

## Decisions

### 视觉风格：科技极简风
**Question**: 展厅呈现什么视觉风格？
**Recommended**: 赛博朋克风（深色背景 + 霓虹发光）
**Chosen**: 科技极简风（白色/浅灰 + 蓝色点缀）
**Rationale**: 用户选择更简洁、专业的风格，类似苹果/特斯拉官网，适合面向客户/领导展示

### 3D 模型来源：程序化生成
**Question**: 可交互 3D 模型的来源？
**Recommended**: 程序化生成（Three.js 几何体组合）
**Chosen**: 程序化生成
**Rationale**: 无需外部模型文件，简化部署，降低加载时间

### 数据来源：静态 JSON
**Question**: 实时数据可视化的数据来源？
**Recommended**: 静态 JSON 数据（无需后端）
**Chosen**: 静态 JSON 数据
**Rationale**: 简化架构，无需后端服务，定时模拟更新即可演示效果

### 交互增强范围
**Question**: 希望增强哪些交互功能？
**Recommended**: 更多交互对象 + 更丰富的展板内容 + 更流畅的控制
**Chosen**: 可交互 3D 模型、实时数据可视化、3D 模型嵌入、全屏沉浸模式、碰撞修复、传送导览、自动巡展
**Rationale**: 综合提升交互体验，满足多场景展示需求
