## 6. 性能要求

### 6.1 加载性能
- 首屏加载时间 < 3 秒
- 3D 模型异步加载，显示加载进度
- 使用 LOD（细节层次）优化远处物体

### 6.2 渲染性能
- 目标帧率：60 FPS
- 使用 requestAnimationFrame 优化渲染
- 避免不必要的重绘

### 6.3 内存管理
- 及时释放不用的纹理和几何体
- 使用对象池复用对象
- 限制同时显示的对象数量

## 7. 部署要求

### 7.1 GitHub Pages 配置
```yaml
# .github/workflows/deploy.yml
name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./
```

### 7.2 访问地址
- 格式：`https://<username>.github.io/<repo-name>/`
- 示例：`https://yourname.github.io/digital-exhibition-hall/`

## 8. 开发规范

### 8.1 代码风格
- 使用 ESLint 进行代码检查
- 使用 Prettier 进行代码格式化
- 命名规范：
  - 类名：PascalCase
  - 函数名：camelCase
  - 常量：UPPER_SNAKE_CASE

### 8.2 注释规范
- 每个类和函数必须有 JSDoc 注释
- 复杂逻辑必须有行内注释
- 模块头部说明模块职责

### 8.3 版本控制
- 使用 Git 进行版本控制
- 提交信息格式：`<type>(<scope>): <subject>`
- 示例：`feat(panel): 添加展板点击交互功能`
